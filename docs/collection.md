# 第六阶段：统一抓取基础设施

本阶段只实现真实联网必需的可靠性边界，不接入具体信息源，也不增加多用户认证、云密钥
管理、容器隔离或新的加密功能。GDELT 和 SEC 分别在步骤 8、9 接入。

## 统一客户端

`SafeHTTPClient` 是后续 SourceAdapter 唯一允许使用的 HTTP 客户端。每个来源必须提供一份
`URLPolicy`，其中明确：

- 允许的域名、协议和端口；
- 允许的 Content-Type；
- 连接、读取和总超时；
- 最大响应字节数和最大重定向次数；
- 请求最小间隔；
- 连续失败阈值和冷却时间；
- 来源自己的 User-Agent。

客户端不会执行 JavaScript，只返回有大小上限的原始 `bytes`。正文后续只能先进入
`RawDocument` 隔离层，不能直接进入证据或报告。

## URL 与网络边界

每次请求及每次重定向都会重新完成：

1. 解析 URL，拒绝凭据、未知协议和未授权端口；
2. 按来源白名单检查精确域名或显式允许的子域名；
3. DNS 解析目标；
4. 拒绝非公网地址，包括环回、私有、链路本地、共享地址和云元数据地址；
5. 检查响应状态、Content-Type、Content-Length 和实际流式读取字节数。

第一版只允许明确白名单的官方或聚合来源，不开放“提交任意 URL”接口。当前实现没有把
解析结果固定到连接层，因此如果未来允许不受信任用户配置域名，必须先增加独立抓取沙箱
或地址固定传输层；个人 Demo 的白名单模式不扩展到该场景。

## 失败与健康状态

来源状态保存在抓取进程内存中：

- 成功后清零连续失败数；
- 超时、DNS 或网络错误达到阈值后，只暂停对应来源；
- 其他来源仍可继续抓取；
- 重启进程后内存状态清零，持久化调度和恢复在步骤 10 实现。

写入 CrawlRun 的失败信息只有固定错误码和是否可重试，不保存目标 URL、查询参数、解析 IP
或底层异常文本。

## 代理

代理只通过本机配置注入统一客户端，适配器不得写死代理：

```text
INVEST_COLLECTOR_PROXY_URL=http://127.0.0.1:7897
```

不需要代理时保持未配置。代理可用性和境外来源连通性会在具体适配器阶段验证。
