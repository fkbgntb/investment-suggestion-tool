# 信息源注册表

信息源注册表保存“从哪里抓、多久抓一次、能抓什么”这些受控配置。它不执行动态代码，
也不允许配置文件指定 Python 模块。只有在应用代码的 `AdapterRegistry` 中登记过的
`adapter_name` 才能保存。目前内置 `mock-rss` 用于测试完整注册流程；真实适配器将在后续步骤注册。

## 配置边界

每个来源包含地区、语言、来源类型、可信等级、HTTPS 基础地址、明确域名白名单、条款链接、
三小时级抓取周期、是否允许全文以及配置版本。基础地址的主机必须出现在白名单中，地址不能
包含用户名或密码。运行抓取时仍会经过统一安全 HTTP 客户端的 DNS、公网地址、重定向、限长和
超时检查，注册成功不代表任意地址都能访问。

来源类型覆盖官方、监管、基金管理人、公司官方、聚合器、媒体、研究、市场数据和社交情绪。
社交与社区来源只能标记为 `SENTIMENT_ONLY`，不能冒充正式证据。可信等级只能通过本机 API
修改，所有创建、更新和停用操作都会写入审计日志。

## 停用与历史数据

`DELETE /api/v1/sources/{source_id}` 是软删除：它把来源设为停用，并将健康状态设为
`DISABLED`。调度器只读取 `/api/v1/sources/schedulable` 返回的来源，因此不会再运行该来源；
数据库中的历史文档、证据和游标不会被删除。

## 健康状态与游标

来源健康状态独立保存，包括连续失败次数、最近成功/失败时间、脱敏错误代码和熔断截止时间。
适配器游标也使用独立表保存，并通过 `state_version` 做乐观并发控制，避免两个任务相互覆盖进度。
这些状态与来源配置分离，因此以后增加 GDELT、SEC、国内交易所或基金公司来源时不需要修改核心表。

本地启动后可在 `/docs` 的 `sources` 分组中创建、查询、更新和停用来源。该组接口只接受回环
地址访问；第一版个人工具中的“管理员”就是运行本机应用的用户，不额外引入账号系统。
