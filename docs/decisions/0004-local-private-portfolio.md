# ADR 0004：本机持仓服务与隐私化分析快照

- 状态：已接受
- 日期：2026-07-19

## 背景

分析工具需要保存用户手工提供的基金持仓和风险边界，但不应接触支付宝凭据，也不能把精确持仓金额发送给外部 AI。后续报告还必须能够解释当时使用的是哪一版持仓。

## 决策

- 持仓、资产和投资档案通过严格领域模型校验后写入工作区数据库；
- 产品类型依据明确事实映射到枚举，不从产品名称猜测；
- 费用数据区分未知、用户提供和官方核验，未知状态禁止携带精确费率；
- 每次分析前生成追加式持仓快照，当前持仓的修改或删除不改变历史快照；
- 外部 AI 只允许接收相对收益、亏损边界使用比例、风险等级和非金额状态；
- 整组持仓 API 只允许本机访问，服务绑定非环回地址时直接禁用；
- CSV 和 OCR 通过独立导入适配器扩展，不进入核心持仓服务。

## 影响

用户需要手工录入或核对持仓，不存在支付宝自动同步。费用和逐笔买入资料不完整时，系统只能提示复核，不能给出精确赎回金额。历史快照会增加少量存储占用，但保证后续报告可复现。
