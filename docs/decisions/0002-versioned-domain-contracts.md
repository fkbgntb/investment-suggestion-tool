# ADR 0002：使用不可变、版本化的领域契约

- 状态：已接受
- 日期：2026-07-19

## 背景

抓取器、市场数据、AI、规则引擎和报告模块会逐步增加。如果它们直接交换字典、数据库行或厂商特有响应，后续替换来源和模型会导致广泛修改，也容易让外部内容或 AI 输出越过安全边界。

## 决定

跨模块数据统一使用 `app/domain` 中的 Pydantic 模型和 Protocol 接口：

- 所有模型固定 Schema 版本、不可变并拒绝未知字段；
- 外部内容和系统控制字段使用嵌套对象隔离；
- 文档状态只能经过显式状态机转换；
- 写入和任务请求携带幂等键；
- AI 不接收存储接口，也不返回更新命令；
- 投资结果始终是建议标签，不是交易指令；
- JSON Schema 作为跨进程和未来多语言实现的机器可读契约。

## 后果

- 第三阶段数据库必须映射这些对象，而不能反向让 ORM 类型污染领域层；
- 新增适配器需要通过同一套契约测试；
- 兼容性修改可以保留 `1.x`，破坏性修改必须发布新主版本；
- 较严格的校验会拒绝未知字段和枚举，调用方需要显式升级。
