# ADR 0003：可迁移且默认安全的本地存储

- 状态：已接受
- 日期：2026-07-19

## 背景

第一版仅供个人在 Windows 本机使用，用户要求所有运行数据保存在 `E:\data`，同时希望以后能迁移到云存储。系统将保存持仓摘要、抓取原文、证据和分析报告，因此需要工作区隔离、可审计迁移、保留策略和可靠备份。

## 决策

- 本地数据库使用 SQLite，访问层使用 SQLAlchemy 2.x；
- 数据库变更只通过 Alembic 版本迁移；
- 所有敏感记录都带工作区边界，父子关系使用复合外键防止跨工作区误连；
- 原始正文默认保留 90 天，过期后保留哈希和必要元数据；
- SQLite 备份先使用在线备份得到一致快照，再压缩并进行认证加密；
- 恢复默认不覆盖现有数据库，且必须通过密文认证和数据库完整性检查；
- 数据库层不提供任何交易、申购、赎回或账户控制结构。

## 影响

SQLite 足以支持当前单用户 Demo，且无需云服务器。SQLAlchemy、仓储接口和 Alembic 迁移降低了未来迁移 PostgreSQL 的成本，但迁移到云端仍需另行完成网络隔离、凭据管理、备份策略和对象存储适配。
