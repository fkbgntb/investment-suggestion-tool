# ADR 0005：使用完整的不可变分类配置版本

## 决定

Topic、Entity、InfluenceRelation 和 Exposure 作为一个完整
`TaxonomyConfiguration` 发布。配置版本只追加，不原地修改；当前活动版本由单独的数据库
指针记录。

## 原因

- 报告需要准确引用生成时所用的配置版本；
- 关键词启停和关系修改必须可回滚、可审计；
- 单条记录原地更新会丢失历史语义；
- 完整快照可以在新增黄金等主题时继续使用同一模型；
- 数据文件经过 Pydantic 校验，不需要执行动态 Python、表达式或插件代码。

## 影响

- 修改一个关键词也要发布新的完整版本；
- 数据量会随版本增加，但个人 MVP 的配置规模很小；
- 旧的 Topic、Entity 和 Exposure 基础表继续保留给后续索引优化，当前可信配置来源以
  `taxonomy_configurations` 快照为准；
- 抓取文档不能调用发布服务，只有本机管理 API 可以改变活动版本。
