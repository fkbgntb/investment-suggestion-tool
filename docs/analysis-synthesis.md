# AI 多空综合

第 16 阶段只综合已验证的结构化证据，不把原始网页、姓名、账户信息、持仓金额或精确加仓
区间发给模型。当前 Prompt 版本为 `analysis-synthesis-1.0.0`。

输入只包含：

- Evidence ID、声明、方向、不确定性、Topic 和影响周期；
- 本地证据总分、来源类型、信任层级和同源转载标记；
- 相对仓位、未实现收益比例、亏损边界使用比例和是否长期持有；
- 第 15 阶段的确定性建议及 `allowed_actions`。

模型输出必须是严格 JSON。所有多方、空方和因果链引用都必须指向输入中真实存在的
Evidence ID；不存在的 ID 会导致本地校验失败。模型建议超出 `allowed_actions` 也会被拒绝，
第 15 阶段的仓位、费率和风险边界不会被 AI 修改。

模型自报置信度不直接采信。系统用所引用证据的确定性分数重新计算上限，并将结果连同
Prompt、模型、输入/输出 token、错误码和时间写入 `analysis_results`。每日综合调用和 token
有独立上限。

没有 DeepSeek API 密钥、超出预算或调用失败时，系统会保留多空证据方向，生成明确标记
`degraded=true` 的规则降级分析，不会伪装成 AI 结论。
