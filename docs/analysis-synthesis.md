# AI 多空综合

第 16 阶段只综合已验证的结构化证据，不把原始网页、姓名、账户信息、持仓金额或精确加仓
区间发给模型。当前 Prompt 版本为 `analysis-synthesis-1.2.1`。

输入只包含：

- Evidence ID、声明、方向、不确定性、Topic 和影响周期；
- 本地证据总分、来源类型、信任层级和同源转载标记；
- 相对仓位、未实现收益比例、亏损边界使用比例和是否长期持有；
- 第 15 阶段的确定性建议及 `allowed_actions`。

模型输出必须是严格 JSON。所有多方、空方和因果链引用都必须指向输入中真实存在的
Evidence ID；不存在的 ID 会导致本地校验失败。模型建议超出 `allowed_actions` 也会被拒绝，
第 15 阶段的仓位、费率和风险边界不会被 AI 修改。

摘要、证据因素、未知项、因果链和失效条件必须使用简体中文；专有名称可以附带官方英文。
每条中文因素必须与同位置的 Evidence ID 一一对应。海外公司事件默认只作为间接行业信号，
除非输入证据明确证明它与当前资产的成分或收入暴露关系。

同一篇文档最多选择一条代表性声明发送给综合模型，防止一篇文章中的多个句子形成虚假共识。
模型自报置信度不直接采信。系统先按文档去重，再用所引用证据的确定性分数重新计算上限；
当方向性证据全部来自二级或聚合新闻时，综合可靠度最高为 `0.35`。该数值表示证据对分析的
支撑程度，不是上涨概率。系统将结果连同
Prompt、模型、输入/输出 token、错误码和时间写入 `analysis_results`。每日综合调用和 token
有独立上限。

网页上的“运行分析”只综合本次新建的 AnalysisRun，不会先消耗预算处理历史积压。定时任务仍可
按批次处理待办记录。调用次数是防止异常循环的辅助上限，token 上限才是本机费用保护的主要边界。

因果链的整体可靠度不得高于其中最弱步骤。若模型只是在该数值上不一致，本地会将整体可靠度
向下收紧到最弱步骤；系统不会向上修改任何可靠度，也不会因此改变证据、方向或建议。

没有 DeepSeek API 密钥、超出预算或调用失败时，系统仍保留证据抽取阶段保存的方向标签，但不会
把标签冒充为已理解的语义因素；报告会生成明确标记 `degraded=true` 的规则降级说明。
