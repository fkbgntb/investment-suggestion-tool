# 自动报告闭环

定时调度现在会消费两类本地任务：

- `process-new-documents`：只有新产生的一级或专业证据才立即生成报告；只有聚合新闻时留到日报。
- `daily-summary`：有新增有效证据时生成当日报告；只有聚合新闻时也只允许生成观察性质报告。

每个任务都会保存明确结果：

- `GENERATED`
- `SKIPPED_NO_NEW_EVIDENCE`
- `SKIPPED_ONLY_AGGREGATOR`
- `SKIPPED_DATA_EXPIRED`
- `SKIPPED_POSITION_INCOMPLETE`
- `FAILED`

任务页面会展示结果和原因。报告幂等指纹包含本地持仓内容、有效证据及动态时效分、决策规则版本和
报告日期；重复任务不会生成第二份相同报告。

分析时会按证据影响周期重新计算时效：

- 短期证据半衰期 3 天；
- 中期证据半衰期 14 天；
- 长期证据半衰期 60 天；
- 未知周期半衰期 7 天。

超过四个半衰期的证据不再进入新报告。历史报告不受影响，仍保留当时使用的证据快照。

## DeepSeek 综合边界

确定性决策仍使用全部有效证据。为了避免大证据集导致模型 JSON 被截断，发送给 DeepSeek 的综合
材料最多包含 16 条代表性证据：优先一级和专业来源，并至少保留主要利多、利空及中性/未知证据。
同一原始文档最多选择一条声明，完整证据列表仍由本地报告保存和展示。

综合输出单独限制为 2400 tokens，并限制因素、未知项、因果链和失效条件的数量。系统会区分输出
截断、结构错误、未知证据引用、越权动作和预算耗尽；任何不合格输出都会被拒绝，再使用带明确原因
的规则降级综合。面向用户的综合字段必须为简体中文。

## 报告嵌入

报告 HTML 只允许同源页面 `http://127.0.0.1:8000` 嵌入。报告内容继续禁用脚本、表单、图片和子
框架；外部网站仍不能把本地报告嵌入自己的页面。

只处理本地积压报告任务、且不访问任何外部信息源：

```powershell
.\.venv\Scripts\python.exe scripts\run_report_tasks_once.py
```

该命令可能在确实需要生成报告且本机已配置 DeepSeek 时产生一次综合分析调用，但不会抓取新闻或
操作任何持仓。
