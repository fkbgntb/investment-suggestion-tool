# 文档清洗、去重与事件聚类

抓取内容先进入 `raw_documents` 隔离区，之后才由确定性的清洗流程写入
`normalized_documents`。报告和后续 AI 模块只能读取清洗后的纯文本，不读取或渲染原始 HTML。

处理流程包括：删除脚本、样式、导航等非正文标签，NFKC Unicode 规范化，移除不可见控制字符，
限制标题、摘要和正文长度，记录异常标记及清洗前后 SHA-256。无法得到有效标题或正文的记录进入
`QUARANTINED`，不会继续分析。

URL 会去除片段和常见跟踪参数并规范化域名、端口和查询参数顺序。规范 URL 或内容哈希相同的记录
被标记为精确重复，但每个来源的文档记录仍然保留。近 72 小时标题与摘要的字符三元组相似度达到
阈值时，文档会加入同一个 `EventCluster`，避免转载数量被误当成独立证据数量。

语言检测通过独立 `LanguageDetector` 接口调用，首版采用无网络、可重复的 Unicode 字符规则；以后可
替换成更准确的模型而不改变清洗服务。手动运行：

```powershell
.\.venv\Scripts\python.exe scripts\run_normalization_once.py
```

正常的三小时定时任务会在发现新文档后自动执行同一流程。
