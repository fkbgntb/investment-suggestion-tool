# 本机代理与 DNS 安全边界

采集器支持显式本机 HTTP 代理，例如：

```text
INVEST_COLLECTOR_PROXY_URL=http://127.0.0.1:7897
```

代理端点必须同时满足：

- 使用 `http`；
- 主机为 `localhost` 或回环 IP；
- 显式提供端口；
- 不包含账号、密码、路径、查询参数或片段。

直连模式会用系统 DNS 解析每个目标域名，并拒绝私有、保留、链路本地和其他非公网地址。启用本机
代理后，目标域名交由代理解析，从而避免 VPN Fake-IP 被误判为真实目标地址；但采集器仍会在每次
请求和重定向时校验 HTTPS、端口及代码白名单域名，IP 字面量中的非公网地址仍会被拒绝。

主要错误含义：

- `DNS_FAILED`：直连模式的系统 DNS 失败；
- `PROXY_DNS_NOT_READY`：本机代理或代理侧域名路由尚未就绪，会短暂重试；
- `ADDRESS_REJECTED`：直连解析到非公网地址，或 URL 直接使用了非公网 IP；
- `TIMEOUT`：已经建立请求流程，但上游响应超时；
- `RATE_LIMITED`：上游明确返回限流；
- `CONTENT_TYPE_REJECTED`：返回内容不符合该来源的允许类型。

工具不会通过关闭 SSRF 检查来解决代理问题，也不会接受远程或带凭据的任意代理地址。
