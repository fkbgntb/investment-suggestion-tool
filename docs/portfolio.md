# 阶段四：投资档案与模拟持仓

## 当前能力

阶段四提供本机使用的投资档案、基金资料和模拟持仓接口。它只维护分析所需数据，不登录支付宝，不读取支付宝账户，也没有申购、赎回或交易执行功能。

当前默认工作区为 `personal-demo`，可以通过本机 `.env` 的 `INVEST_PORTFOLIO_WORKSPACE_ID` 修改。数据仍保存到 `E:\data\investment_tool.db`。

## 初始化 007300 示例

```powershell
.\.venv\Scripts\python.exe scripts\storage_admin.py migrate
.\.venv\Scripts\python.exe scripts\seed_demo_portfolio.py
```

示例脚本只在记录不存在时创建：

- 个人训练档案：组合亏损边界 1,000 元、单基金预警 200 元、复核线 300 元；
- 基金 `007300`：产品类型明确记录为 `ETF_LINK`，跟踪场内 ETF `512480`；
- 2026-07-17 持仓快照：157.89 份、成本 640 元、当时价值 653.59 元、每月定投 50 元；
- 最早和最近买入日期暂按 2026-04-01 与 2026-07-01 记录；
- 逐笔买入批次和官方费用尚不完整，因此明确标记为未知，不据此输出精确赎回金额。

脚本再次执行时不会覆盖你之后手工修改的内容。

## 本机 API

启动服务：

```powershell
.\scripts\run-dev.ps1
```

访问 `http://127.0.0.1:8000/docs`，使用 `portfolio` 分组：

- 录入和修改投资档案；
- 录入和修改基金资料；
- 录入、查询、修改和删除模拟持仓；
- 根据明确的产品事实识别 `ETF`、`ETF_LINK`、`INDEX_FUND` 或 `UNKNOWN`；
- 在分析前生成不可变持仓快照；
- 从快照生成允许发送给 AI 的隐私化风险摘要。

这些接口只接受环回地址客户端，并拒绝非本机 Origin。即使显式允许服务绑定 `0.0.0.0`，持仓接口也会禁用。普通访问日志只包含请求方法、路径和状态码，金额不会放入 URL。

## 费用与持有期

费用数据具有三种状态：

- `UNKNOWN`：没有可靠费用，不允许同时填写费率；
- `USER_PROVIDED`：用户依据手头材料录入，必须注明来源和生效日期；
- `OFFICIAL_VERIFIED`：已从官方资料核验，必须保存来源 URL、核验时间和生效日期。

赎回费采用不重叠的持有天数区间。逐笔买入批次可记录购买日、确认日、份额和成本；资料不完整时，`holding_period_data_complete` 必须保持 `false`。

## 分析快照与 AI 隐私边界

分析快照是追加式记录。更新或删除当前持仓不会修改已经生成的快照，后续分析运行可以通过 `position_snapshot_id` 引用它。

允许离开本机的 AI 风险摘要只包含：

- 资产 ID 和产品类型；
- 相对收益率；
- 已使用的亏损边界比例和风险等级；
- 定投是否启用；
- 持有天数、批次完整性和费用数据状态。

摘要不包含档案名称、支付宝账号、持有份额、成本金额、当前金额、可用资金或精确加仓资金。当前阶段只生成摘要，不调用 DeepSeek。

## 后续导入扩展

领域层已经预留独立的 `PortfolioImportAdapter`，未来 CSV 或 OCR 导入必须先在本机解析和校验，再调用同一套服务。导入适配器不会获得交易权限，也不能绕过未知字段拒绝、金额校验和工作区隔离。
