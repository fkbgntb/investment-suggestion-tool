# 质量评估与影子运行

第 20 阶段用于验证个人 Demo 在异常信息、模型失败和网络失败时能安全降级。影子运行只生成建议与审计记录，不登录支付宝、不下单，也不会自动申购或赎回。

## 完整质量门槛

```powershell
.\scripts\check.ps1 -Audit
```

该命令执行静态检查、格式检查、密钥扫描、Schema 漂移检查、完整测试与覆盖率门槛，以及联网依赖漏洞扫描。测试集覆盖利多、利空、多空冲突、无关/重复/过期内容、提示注入、非法 JSON、未知 Evidence ID、SSRF、重定向、超大响应、CSRF、任务恢复、AI 预算耗尽和数据库恢复。

## 执行一次影子运行

确保本机 `.env` 已配置 `E:\data`、组合参考金额，并已初始化示例持仓，然后执行：

```powershell
.\.venv\Scripts\python.exe scripts\run_shadow_once.py
```

运行顺序为：三小时增量抓取、清洗与去重、相关性判断、证据抽取与评分、确定性决策、受限 AI/规则综合、报告生成、质量指标统计。即使一个来源或 DeepSeek 不可用，其余流程仍会继续，并把结果标记为 `DEGRADED`。

每次结果保存为 `E:\data\shadow-runs\<shadow_run_id>.json`，`latest.json` 指向最近一次快照。快照只包含建议、版本、计数、比率和变化原因，不包含 API 密钥或精确持仓金额。原始历史文件不会被覆盖；规则或提示词发生变化时，版本号也会随记录保留。

当前只统计模型 token，不虚构人民币费用。只有在明确配置并版本化供应商计费规则后，`estimated_cost_cny` 才能有数值。

首次运行只能证明闭环可工作，不能证明建议有效。投入真实决策前，应持续观察多次报告、人工打开来源链接核对证据，并确认自己能理解“不确定信息”和“失效条件”。
