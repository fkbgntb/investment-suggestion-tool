## 1. 第一版目标

第一版只完成一个围绕半导体主题的个人投资分析 Demo，不接入黄金、中证 500、沪深 300、人工智能等其他主题。

Demo 需要跑通以下最小闭环：

1. 手工保存一只半导体基金的持仓快照；
2. 每 3 小时检查 GDELT 和 SEC/代表性公司正式披露；
3. 对新信息进行安全清洗、去重和半导体主题相关性筛选；
4. 使用 DeepSeek 兼容的 AIProvider 抽取结构化证据；
5. 使用确定性规则约束建议强度，避免 AI 给出过激或证据不足的建议；
6. 生成带多空证据、未知项、置信度、来源和失效条件的个人分析报告；
7. 只向用户提供分析建议，由用户独立判断；系统不设计任何申购、赎回或交易执行能力。

### 4.1 第一版风险规则草案

在最大可接受亏损尚未确定前，采用保守规则：

- AI 不允许直接给出金额；
- 单次新增申购不超过 200 元；
- 100 元作为允许加仓时的默认建议上限，不代表自动执行；
- 每月 50 元定投作为当前有效计划记录；
- 累计亏损达到 200 元时在报告中显示风险预警；
- 累计亏损达到 300 元时强制重新分析，并优先建议用户复核是否暂停加仓；
- 全部 3,000 元最多亏损约 1,000 元作为未来组合级参数，MVP 不用它直接控制 `007300`；
- 只有一个聚合新闻来源时不得触发加仓或赎回；
- 数据过期、来源失败或证据冲突严重时返回“继续观察”；
- 第一版不要求用户凭经验设置固定赎回金额；
- 给出降低仓位的建议前必须检查买入批次、持有期和官方赎回费；
- 在赎回费数据和批次信息不足时，只能输出“建议复核是否降低仓位”，不输出精确赎回金额；
- 用户始终独立判断是否采纳建议；系统没有交易适配器、申赎执行接口或账户控制能力。

## 5. 第一版信息范围

### 5.1 主题范围

只关注与本试验基金相关的半导体产业链：

- 半导体、芯片、晶圆；
- DRAM、NAND 和存储价格；
- 半导体设计、制造、设备和材料；
- 库存、产能利用率和资本开支；
- 手机、PC、服务器和消费电子销量；
- 铜等可能影响终端产品成本和需求的商品；
- 与目标指数成分和代表性全球公司有关的正式披露。

### 5.2 示例来源

| 来源 | 第一版用途 | 可信边界 |
|---|---|---|
| GDELT | 发现全球多语言相关新闻 | 只作为发现层；默认不抓任意返回链接全文 |
| SEC/代表性公司披露 | 获取高可信度正式披露 | 正式事实不自动等同于对本基金的直接影响 |

第一版暂不增加社交媒体、收费研报、国内全网新闻和任意网页全文抓取。

### 5.3 更新与 AI 触发

- 抓取周期：每 3 小时；
- 没有新文档：不调用 AI；
- 有新文档但规则判定无关：不调用综合分析；
- 有新相关证据：更新证据库并触发一次综合分析；
- 每日 AI 调用量和费用必须有硬上限；
- 数据连续 6～8 小时未更新：报告显示“数据可能过期”。

## 6. AI 技术基线

### 6.1 提供商

| 参数 | 当前决定 |
|---|---|
| AIProvider | DeepSeek |
| 使用方式 | API，用户已确认具备 API 密钥 |
| 证据抽取模型 | 通过环境变量配置，优先使用成本较低且支持 JSON 输出的模型 |
| 综合分析模型 | 通过环境变量配置，不在代码中写死 |
| API 密钥 | 只能放在本机环境变量或后续密钥管理服务中，不进入对话、Git、日志或数据库 |

DeepSeek 当前官方 API 文档显示模型支持 JSON Output，并按 Token 计费。模型名称和价格可能变化，因此代码只读取配置，不固定依赖旧模型别名：

- 官方价格页：https://api-docs.deepseek.com/zh-cn/quick_start/pricing
- API Base URL：https://api.deepseek.com

### 6.2 成本控制草案

第一版不立即承诺具体月费用，先采用硬性预算机制：

- 默认不在没有新证据时调用 AI；
- 每篇文档先用规则筛选；
- 单篇文档设置最大输入长度；
- 每日设置文档数、AI 请求数和 Token 上限；
- 页面显示当日和当月使用量；
- 达到预算后停止 AI，保留原始文档并返回“预算已用尽”；
- 在正式充值前，用固定样例测量一次实际 Token 消耗，再确定月度预算。

### 6.3 安全限制

- 新闻正文属于不可信内容；
- 证据抽取模型没有 Web、代码执行、数据库写入或其他工具权限；
- 综合模型只接收结构化证据，不读取原始网页；
- 精确持仓金额不需要发送给外部模型，优先发送仓位比例、相对收益和建议边界；
- AI 不能绕过风险规则生成证据不足或强度过高的建议；
- JSON Schema 校验失败时不生成报告。

```text

+ SQLite
```

- 本地网络无法访问的境外来源可能抓取失败；
- 第一版需要提供手动刷新和来源失败提示；
- 代码结构仍按可迁移到 Docker/Linux/云服务器设计；
- Demo 跑通后再比较云服务器、托管任务平台或境内外双节点。

## 8. 安全与隐私基线

- 不保存支付宝账号、密码、订单号、身份证、银行卡或二维码；
- 不把用户截图加入 Git 仓库；
- 只保存完成分析所需的持仓数字；
- API 密钥不写入代码和 `.env.example`；
- 本地服务默认只监听 `127.0.0.1`；
- 第一版不开放公网；
- GDELT 返回链接默认不自动抓取全文；
- 只有白名单官方域名可以抓取正文；
- 外部内容进入 AI 前经过清洗、限长和提示注入隔离；
- 所有报告保留证据 ID、规则版本、Prompt 版本和模型名称；
- 没有交易模块、交易接口或账户控制能力。
