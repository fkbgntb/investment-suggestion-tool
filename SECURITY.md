# Security Policy

## Scope

当前项目仅供单个用户在本机使用。它处理持仓和外部新闻等敏感或不可信数据，但不处理支付宝凭据，也不具备任何交易执行能力。

## Security invariants

- 不将真实 API 密钥提交到 Git；
- 不将支付宝截图、账号、订单号、身份证或银行卡信息写入仓库；
- 默认仅监听本机地址；
- 外部内容必须经过安全抓取和清洗后才能进入 AI；
- AI 不获得工具、数据库写入或账户控制能力；
- 交易、申购和赎回只能作为文本建议，不得实现执行接口；
- 数据不足或校验失败时安全拒绝生成新建议。

## Local secret handling

真实密钥仅允许保存在被 Git 忽略的 `.env` 或操作系统环境变量中。日志、异常响应、测试夹具和文档不得包含真实密钥。
